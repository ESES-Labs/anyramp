Selamat Datang di Ekosistem H2H Neticon Pay
Neticon Pay Host-to-Host (H2H) adalah infrastruktur API RESTful berstandar enterprise yang dibangun khusus untuk mengakselerasi ekspansi pengembang aplikasi (developer), pemegang server PPOB, serta para penggiat bisnis digital di Indonesia. Infrastruktur ini memungkinkan sistem Anda untuk terhubung langsung dengan basis layanan kami secara instan, mengautomasi pemesanan tanpa intervensi pihak ketiga.
Arsitektur Pemotongan Saldo Terpusat
Implementasi API menggunakan Saldo Akun Neticon Pay Mitra sebagai satu-satunya sumber pendanaan (deposit-based).
Alur Transaksional: Pengguna akhir (end-user) bertransaksi melalui aplikasi Anda menggunakan modul pembayaran Anda sendiri. Setelah memvalidasi pelunasan dari sisi Anda, antarmuka backend Anda akan mengirim instruksi pemrosesan produk kepada kami. Seketika itu pula, Neticon Pay akan mengurangi saldo Anda sejumlah Harga Dasar (Harga Modal) mitra, memberikan Anda kendali mutlak atas margin laba/profitabilitas.
Komitmen Infrastruktur Neticon Pay:
Pelaporan Real-Time (Webhook Asynchronous): Menghilangkan kelemahan latensi metode pemeriksaan tradisional (Polling) yang merugikan. Server Neticon akan melempar transmisi pelaporan HTTP (Webhook) saat status di penyedia layanan pusat dinyatakan SUKSES/GAGAL.
Protokol Keamanan Tingkat Tinggi (Dynamic MD5 Sign): Sistem perisai anti maling. Memblokir skenario manipulasi data dan Replay Attacks dengan mengacak hash kunci referensi yang berganti-ganti di tiap detik transaksi.

Protokol Kriptografi Tanda Tangan (Signature)
Demi melindungi pilar validitas dan integritas transaksi, setiap bundel pemuatan format JSON mewajibkan transmisi parameter identifikasi sign. Atribut hash ini mengevaluasi otorisasi tingkat pengguna internal untuk menangkal intersepsi komunikasi.
Skema Kriptografi Endpoint Inquiry (Profil/Katalog)
md5(user_id + api_key + action)
Lakukan proses String Concatenation pada instrumen variabel user_id, api_key, serta nilai tipe eksekusi action (contoh spesifik: "profile"), selanjutnya implementasikan formula hashing MD5 128-bit.
Skema Kriptografi Endpoint Transaksional (Pembelian/Status)
md5(user_id + api_key + ref_id)
Sebagai respon perlindungan berlapis di sisi transaksional, Signature wajib direkatkan pada kode unik pemesanan Anda (ref_id). Algoritma ini me-nullifikasi fungsi identitas hash bila dipergunakan ulang (Replay Attack).
Database Katalog Harga
Unduh file Excel/CSV ini untuk memasukkan data (Kode SKU & Harga) secara manual ke sistem Anda. Data otomatis disinkronisasi dengan harga jual terkini.

Sinkronisasi Inventori (Via API)
Akses fungsional endpoint "action": "pricelist" memampukan sistem mesin server internal milik Mitra untuk mendisplay varian Stock Keeping Unit (SKU), ketersediaan stok/status produk, beserta acuan struktur Pricing Modal Neticon Pay untuk kepentingan markup harga ritel aplikasi Anda.
Protokol Praktik Terbaik (Best Practice): Sistem Anda sangat diwajibkan untuk memberlakukan manajemen Data Caching Strategy. Gunakan fungsi Cron Job Server untuk menarik data Pricelist per jam. Tidak diperkenankan me-request API Pricelist secara konstan bersamaan lalu-lalang klik pengguna yang dapat memicu pemblokiran otomatis dari sistem pertahanan DDoS kami.

Metodologi Eksekusi Permintaan (Order Placement)
Inisiasi param "action": "order" mendelegasikan otoritas penuh pada arsitektur mesin backend Neticon Pay untuk segera mengalokasikan validasi pemotongan saldo finansial, dan mengirimkan transmisi penagihan kepada aggregator pusat telekomunikasi.
Komponen Variabel Mandatori POST:
target
Identitas presisi dari objek tujuan (Contoh: Parameter MSISDN Nomor Seluler Pelanggan, User ID Entitas Ekosistem Game, ID Meteran Pratam PLN).
kode_produk
Pemilihan Identitas Kode Produk / SKU referensial unik (misal: S10, FF50, DANA20) bersumber dari katalog data.
ref_id
Parameter Key Index tunggal berbasis referensi penomoran Invoice lokal milik Anda. Aturan kembar ganda ditolak absolut untuk mencegah redundansi berbayar (Double Purchase). Limitasi maksimum diatur dalam rentang 50 Karakter Alfanumerik.
*callback_url
*(Instrumen Optimasi Sistem Asinkron)* Titik akhir spesifik pendaratan balasan (Endpoint URL Server Tujuan) sebagai destinasi integrasi penerimaan reportase status secara utuh dan Real-Time.

Notifikasi Asinkron & Pelaporan Webhook
Mekanisme transmisi Callback / Webhook dirancang secara komprehensif sebagai jalur pemberitahuan informasi pemrosesan mutlak (Event-Driven Reporting). Metode ini secara harfiah menghapus rutinitas pertanyaan frekuensi repetitif (Polling) server yang membebani jalur transfer data interkoneksi kedua pihak. Sistem agregasi Neticon Pay akan melakukan HTTP POST Call secara proaktif mengeksekusi Ping berisikan balasan fungsional JSON (Success / Failed) secara mendetail mengenaik kepastian injeksi produk ke nomor penerima pada parameter callback_url saat status dinyatakan final.

Implementasi Dasar Komputasi Klien Penadah Endpoint:
Validasi Otorisasi Algoritma Kriptografi HMAC (Wajib): Identifikasikan keabsahan laporan payload melalui dekripsi nilai balikan sign dengan menggenerasikan ulang rumusan: md5(ref_id + api_key_mitra + status). Validitas yang berpadu mengindikasikan koneksi bersifat autentik, tertutup, dan terlindungi dari eksploitasi peretas eksternal.

Model Sinkronisasi Progresif Konvensional (Status Polling)
Bila konfigurasi sistem internal platform Anda tidak merespon/mendukung transmisi balik asinkron, kami menyediakan mekanisme pengecekan status yang dilakukan mandiri. Gunakan "action": "status" disertai korelasi id referensi yang valid pada skema variabel pemanggilan. Diwajibkan untuk memberlakukan penundaan batas wajar durasi interaksi pemeriksaan bersyarat setidaknya tiap 10 s/d 15 detik antar permohonan.
Sintaksis Autentikasi Menggunakan Parameter Eksekusi 2: md5(user_id + api_key + ref_id)
Komponen Permintaan Penelusuran (JSON POST):

"action": "status",
"sign": "md5(1NP-1234567890TRX-001)",

Modul Injeksi Perangkat Lunak Bot Otomatis
Konfigurasi arsitektur modular yang disediakan mempermudah akselerasi integrasi mesin Chatbot Anda dengan infrastruktur Neticon Pay. Dirancang untuk efisiensi development aplikasi basis Node.js (seperti Library Baileys, Whatsapp-web.js, atau ekosistem Telegraf pada Telegram), model logika ini disertai mekanisme Dynamic Hashing Protection yang menyusun sandi validasi MD5 mengikuti parameter operasional yang dibutuhkan secara mandiri.
Wrapper Klien Pemrosesan
Salin Kode
const axios = require('axios');
const crypto = require('crypto');
// --- 1. OTENTIKASI & KONFIGURASI KREDENSIAL ---
const NETICON_URL = "https://m.neticonpay.my.id/api/h2h/api.php";
const USER_ID = "ISI_USER_ID_ANDA";
const API_KEY = "ISI_API_KEY_ANDA";
// --- 2. ASYNCHRONOUS EXECUTABLE FUNCTION (DYNAMIC SIGNING V2.1) ---
async function tembakNeticon(action, params = {}) {
let stringToHash = "";
// Proteksi Integritas dengan Konversi Pola Hash Logikal
if (action === 'order' || action === 'status') {
if(!params.ref_id) return { status: false, message: "Kebutuhan entitas ref_id tidak terpenuhi." };
stringToHash = USER_ID + API_KEY + params.ref_id;
} else {
stringToHash = USER_ID + API_KEY + action;
}
// Melakukan konversi format string (Heksadesimal Encryption Checksum)
const sign = crypto.createHash('md5').update(stringToHash).digest('hex');
const payload = {
user_id: USER_ID,
api_key: API_KEY,
action: action,
sign: sign,
...params
};
try {
const response = await axios.post(NETICON_URL, payload);
return response.data;
} catch (error) {
return { status: false, message: error.message };
}
}
