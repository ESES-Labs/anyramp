Dokumentasi API H2H
Selamat datang di Dokumentasi Resmi API Host-to-Host (H2H) Neticon Pay. Infrastruktur API kami dirancang secara profesional untuk membantu Anda mengotomatisasi pembuatan tagihan QRIS Dinamis serta memvalidasi pembayaran secara real-time langsung ke dalam ekosistem sistem Anda (Website/Aplikasi).
Sinkronisasi Saldo Terpusat (My Qris)
Kami menjamin keamanan tingkat tinggi untuk setiap aset Anda. Seluruh dana dari transaksi H2H yang berhasil divalidasi melalui API ini akan secara otomatis masuk dan terakumulasi secara utuh ke dalam saldo aplikasi My Qris Anda. Anda dapat memantau mutasi, mengunduh laporan audit, serta melakukan pencairan dana kapan saja melalui satu pintu dashboard utama yang aman.
Base Endpoint URL
https://qris.neticonpay.my.id/qris.php

Pedoman Keamanan Wajib
Dilarang Keras Auto-Approve Saldo!
Anda TIDAK BOLEH menambahkan saldo atau memproses pesanan user Anda hanya karena user menekan tombol "Saya Sudah Bayar" di aplikasi/web Anda. Manipulasi di sisi Frontend (seperti Inspect Element) sangat mudah dilakukan oleh oknum yang tidak bertanggung jawab.
Sistem Anda wajib menahan pesanan sampai Server Anda melakukan pengecekan lintas server (Server-to-Server) ke Neticon Pay melalui Endpoint Validasi, dan mendapatkan balasan mutlak "status": "success".
Simpan API Key di Backend
Jangan pernah menaruh API Key Anda di dalam kode HTML, JavaScript (Frontend), atau aplikasi mobile sisi klien. API Key hanya boleh dieksekusi melalui skrip Backend (PHP, Node.js, dll).
Aktifkan IP Whitelist
Daftarkan IP Server/Hosting website Anda di dashboard My Qris (Menu Integrasi H2H). Request dari IP yang tidak terdaftar akan otomatis ditolak oleh sistem pengamanan kami.
POST
1. Buat Tagihan Baru
Langkah pertama dalam alur H2H adalah menginisiasi transaksi. Sistem kami akan mencatat nominal (wajib ditambah kode unik dari sistem Anda) dan merespons dengan ID Transaksi (Trx ID).
Parameter Request
Parameter
Tipe
Deskripsi
action
String
Harus diisi dengan request_deposit.
api_key
String
Secret API Key H2H Anda.
user_id
String
Merchant Code / User ID akun Neticon Anda.
amount
Integer
Total nominal tagihan. PENTING: Sistem Anda harus menambahkan 3 angka unik di belakang nominal (contoh: 50123). Minimal nominal Rp 1.000.
Request Data (JSON)
Salin
{
"action": "request_deposit",
"api_key": "NP_1234567890abcdef",
"user_id": "MERCHANT_01",
"amount": 50123
}
Response Diterima
Salin
{
"result": true,
"trx_id": "H2H1710001234",
"amount": 50123
}
* Simpan `trx_id` yang didapat ke dalam database sistem Anda untuk keperluan pengecekan status.

2. Validasi Pembayaran (Wajib)
Ini adalah tahap krusial. Sistem (Backend) Anda harus menembak endpoint ini untuk mengecek apakah pelanggan sudah mentransfer dana yang sesuai. Lakukan pengecekan secara polling (misal: setiap 5-10 detik) atau saat user menekan tombol "Cek Pembayaran".

Response Sukses (Valid)
Salin
{
"result": true,
"status": "success"
}
Alur Logika di Sistem Anda:
Jika response.status == "success" {
// AMAN! Dana sudah terverifikasi di Neticon Pay.
// Update database Anda: Ubah pesanan jadi LUNAS.
// Berikan hak akses / saldo ke user Anda di sistem Anda.
} else {
// Jangan lakukan apapun. Pembayaran belum valid.
}

Kombinasi API Key & User ID salah, atau Akun Anda sedang diblokir.
